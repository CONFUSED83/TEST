Ch 1: Variables & Data Types

Variables are reserved memory locations to store values. Unlike other languages, Python is dynamically typed.

What is Dynamic Typing?

It means you don't declare the type of a variable when you assign a value to it. The interpreter figures out the type at runtime.

$$ \text{Syntax: } \mathbf{\text{variable\_name} = \text{value}} $$

To check the type of a variable, use the type() function. The following variables will have the types listed:

  • x = 10 (int)
  • y = 3.14 (float)
  • z = "Hello" (str)
  • b = True (bool)

String Formatting (f-strings)

F-strings (formatted string literals) provide a concise, readable way to embed Python expressions inside string constants. They start with the letter f.

Example:

$$\mathbf{a} = 10, \quad \mathbf{s} = \text{"Value is: } \{ \mathbf{a} \} \text{"} \quad \implies \quad \mathbf{s} = \text{"Value is: 10"} $$

String Indexing & Slicing

Strings are sequences, indexed from 0. Negative indices count backward from -1.

$$\text{str[start:stop:step]}$$
  • "Python"[0] $\implies$ 'P'
  • "Python"[-1] $\implies$ 'n'
  • "Python"[1:4] $\implies$ 'yth' (stop index is exclusive)
  • "Python"[::-1] $\implies$ Reverse!
Ch 2: Operators

Arithmetic Operators

Standard math operators, plus a few key ones:

  • **: Exponentiation (e.g., $2^{3} = 8$).
  • //: Floor Division (returns the integer part of the quotient).
  • %: Modulo (returns the remainder).
$$ 10 \text{ // } 3 = 3 \quad \text{and} \quad 10 \text{ \% } 3 = 1 $$

Logical Operators (Boolean Algebra)

Used to combine conditional statements. Remember the order of precedence: $\mathbf{\text{not}} \to \mathbf{\text{and}} \to \mathbf{\text{or}}$.

  • and: Both sides must be True.
  • or: At least one side must be True.
  • not: Inverts the boolean value.
Ch 3: Control Flow (Indentation!)

The `if`/`elif`/`else` Block

The core of decision-making. Indentation (4 spaces) is mandatory and defines the scope of the block.

$$ \text{if condition}_1: $$ $$ \quad \text{code block for condition}_1 $$ $$ \text{elif condition}_2: $$ $$ \quad \text{code block for condition}_2 $$ $$ \text{else}: $$ $$ \quad \text{default code block} $$

Loops: `for` and `while`

  • for: Used for iterating over a sequence (list, tuple, string, range).
  • while: Used for looping as long as a condition remains true. Must manage the loop variable to avoid infinite loops.
Ch 4: Functions (DRY)

Functions are reusable blocks of code that perform a specific task (DRY: Don't Repeat Yourself). Defined using the def keyword.

$$ \text{def } \mathbf{\text{function\_name}(\text{param}_1, \dots)}: $$ $$ \quad \text{DOCSTRING (optional)} $$ $$ \quad \text{code to execute} $$ $$ \quad \mathbf{\text{return } \text{value}} \text{ (optional)} $$

Function Arguments

Arguments can be passed by position or by keyword (e.g., func(a=1, b=2)). Keyword arguments allow for defining default values.

The Lambda Function is a small anonymous function (not defined with def).

$$ \text{lambda args: expression} $$
Ch 5: Data Structures

Lists $\ne$ Tuples

  • Lists: Mutable (can be changed). Defined with square brackets $\mathbf{[ ]}$.
  • Tuples: Immutable (cannot be changed after creation). Defined with parentheses $\mathbf{( )}$.
$$\text{List: } L = [1, 2, 3] \implies L[0] = 99 \text{ is OK}$$ $$\text{Tuple: } T = (1, 2, 3) \implies T[0] = 99 \text{ is } \mathbf{\text{Not OK!}}$$

List Comprehensions

A concise way to create lists. Extremely Pythonic and fast.

$$\mathbf{\text{new\_list}} = [\mathbf{\text{expression}} \text{ for } \mathbf{\text{item}} \text{ in } \mathbf{\text{iterable}} \text{ if } \mathbf{\text{condition}}]$$

Dictionaries (Key-Value)

Unordered collection of Key: Value pairs. Keys must be unique and immutable (e.g., strings, numbers, tuples).

$$\mathbf{\text{dict}} = \{ \mathbf{\text{key}_1: \text{value}_1}, \mathbf{\text{key}_2: \text{value}_2} \}$$
  • Accessing: my\_dict['key']
  • Adding/Updating: my\_dict['new\_key'] = 'new\_value'

Sets

Unordered collection of unique elements. Used for membership testing and eliminating duplicates.

$$\text{Set Operations: } A \cup B \quad (\text{Union}), \quad A \cap B \quad (\text{Intersection})$$
